Virtual Engineering Pipeline

How a task goes from idea to live

An AI-powered engineering pipeline that handles the heavy lifting. Here's what happens at each step and who's involved.

1
Your Team
Drop a task into ClickUp
Describe what you need built or fixed. Include what "done" looks like — the clearer the description, the better the result. Add any relevant links, screenshots, or assets.
Just like assigning work to a teammate ↗
2
Oscar — AI Project Manager
The PM picks it up and plans the work
Oscar, the AI project manager, reads the task, figures out what kind of work it is (design, backend logic, server setup, etc.), and assigns it to the right AI specialist. If the task is big, it splits it into smaller pieces that can be worked on at the same time.
💬 You'll see a Slack message: "Starting [task]: assigned to Luna. Estimated time: 2 hours."
3
AI Specialists — Luna, Marcus, Rex
The work gets done
Specialized AI workers write the code. Luna handles the visual side (dashboards, pages), Marcus builds the behind-the-scenes logic (APIs, data), and Rex manages servers and deployment. They can work in parallel, so multiple things happen at once.
⏱ Most tasks complete in 1–4 hours
Automated quality checks begin
4
Vigil + Quinn — Security & Quality Review
Two layers of automated checks
Before anyone sees the work, it goes through two automated reviews. Vigil (security check) scans for anything sensitive that shouldn't be in the code — passwords, keys, known vulnerabilities. Quinn (quality check) makes sure the code actually does what was asked and doesn't break anything else.
If issues are found, the work is automatically sent back for fixes — up to two times. If it still can't pass, the Tech Lead gets notified.
Think of it as two editors proofreading before it reaches you
5
Rex — Automatic Deployment
Deployed to a test environment
Once the automated checks pass, Rex automatically puts the new feature on a staging site — a private copy of the app where you can test things without affecting the real product. The system verifies the site loads correctly and nothing is broken.
🔗 You'll see a Slack message with a link to preview the changes on the staging site.
6
Tech Lead (Human)
Human review and approval
The Tech Lead opens the staging link, tests the feature, and reviews the code. Nothing goes live without human approval. If changes are needed, it goes back for another round. If it looks good, they tell Oscar to merge and deploy.
✓
Done
Live in production
Oscar merges the code, Rex deploys it to production, and it's live. ClickUp automatically marks the task as complete. On to the next one.
